Серіалізація та десеріалізація, Маршаллінг та демаршаллінг - це поняття, які використовуються в програмуванні для
збереження об'єктів в певному форматі (зазвичай для збереження на диск або передачі через мережу) та їх відновлення
у вихідний об'єкт. Вони допомагають зберегти стан об'єкта і змогу передавати його між різними системами або зберігати
для подальшого використання.

Серіалізація - це процес перетворення об'єкта в певний формат (наприклад, рядок), який можна зберегти або передати.
У Java це реалізується інтерфейсом Serializable.

Десеріалізація - це процес відновлення об'єкта з серіалізованого формату. В Java це також реалізується за допомогою
інтерфейсу Serializable.

Маршаллінг - це термін, який часто використовується у контексті обробки даних у форматах, таких як XML або JSON.
Він означає перетворення об'єкта в рядок або байтовий потік відповідно до структури цього формату.

Демаршаллінг - це процес відновлення об'єкта з рядка або байтового потоку, який був попередньо створений через
маршаллінг.

XML (Extensible Markup Language) - це мова розмітки, яка використовується для представлення даних у структурованому
форматі. Головна різниця між XML і HTML полягає в тому, що XML не визначає, як виглядає вміст (текст на веб-сторінці),
а лише вказує структуру даних. HTML, навпаки, використовується для відображення вмісту веб-сторінок і містить визначення
для вигляду та поведінки елементів на сторінці.

Деякі відомі парсери XML в Java:

DOM (Document Object Model) - це API, яке розглядає XML-документ як ієрархію об'єктів (вузлів дерева DOM) і дозволяє
змінювати структуру та вміст документа. DOM вимагає завантаження всього документа в пам'ять, тому він може бути
неефективним для великих XML-файлів.

SAX (Simple API for XML) - це подійний парсер, який обходить XML-документ і викликає методи зареєстрованих обробників
для обробки знайдених елементів. SAX працює в потоковому режимі і не вимагає завантаження всього документа в пам'ять,
що робить його ефективним для обробки великих файлів.

JSON (JavaScript Object Notation) - це легкий формат обміну даними, який використовується для представлення
структурованих даних в текстовому форматі. JSON використовується для обміну даними між веб-серверами і клієнтами,
а також для зберігання конфігураційних даних та інших структурованих даних в програмах.

JAXB (Java Architecture for XML Binding) - це технологія в Java, яка дозволяє перетворювати між об'єктами Java і
XML-документами і забезпечує автоматичну серіалізацію та десеріалізацію. JAXB надає анотації для класів, щоб вказати,
які елементи XML відповідають полям об'єкта. Вона допомагає спростити роботу з XML в Java-програмах.
